Author-Facing Rendering Hints
----------------------------
- Diagnostics separate authored logical style, backend capability, renderer policy, and final backend realization so authors can see where a visual difference was introduced.
- Tri-state logical fields now matter in diagnostics: unspecified means the field was absent, explicitly enabled means the author requested it on, and explicitly disabled means the author intentionally forced it off.
- Runtime diagnostics recorded only direct realizations for the style features encountered during this session.
- Some authored style features were realized directly rather than conservatively omitted, which reflects the renderer's shift toward maximal safe feature use when the capability/policy pair allows it.

Notes
-----
- Diagnostics describe renderer behavior only.
- Logical Style data stored in ScreenBuffer is not mutated by diagnostics or adaptation.
- Tri-state logical reporting exists to distinguish absent fields from explicit enable/disable intent.
- Host selection and renderer selection are reported separately on purpose.
- VT enablement/availability and active renderer path are reported separately on purpose.
- Output differences between authored style and visible result may be caused by downgrade, omission, approximation, or deferred emulation.
- Color adaptation reasons are reported from ColorResolver decisions rather than recreated in the renderer.
- Bright basic color capability describes palette/intensity color presentation, not bold or dim text semantics.
- Author-facing hints are advisory only and do not change rendering behavior.
